# 第 07 章 API 与自己部署

> 用 AI 有两种方式：通过网络"租用"（API/订阅），或者把 AI "搬回家"（本地部署）。这一章帮你搞清楚它们的区别，知道哪种适合你。

---

## 7.1 两种方式的比喻

想象你需要用电：

| 方式 | 类比 |
| --- | --- |
| **API / 订阅** | 从电网买电——用多少付多少，不用自己建电厂 |
| **本地部署** | 自己装发电机——前期投入大，但用电完全自主 |

---

## 7.2 API：通过"接口"使用 AI

### 什么是 API

**API（Application Programming Interface）**，中文叫"应用程序接口"。

你可以理解为：AI 公司开了一个"窗口"，你的程序通过这个窗口提交问题、拿到答案。

**使用流程**：
1. 注册账号，获得一个"密钥"（API Key）
2. 你的程序把问题发到这个"窗口"
3. AI 在云端处理完，把答案发回来
4. 按处理的 Token 数量计费

> 💡 如果你只是在网页/App 上聊天（如 ChatGPT 网页版），其实也是在用 API——只不过"窗口"已经封装好了，你感觉不到。

### 两种付费方式

| 方式 | 说明 | 适合谁 |
| --- | --- | --- |
| **订阅制** | 每月固定费用，不限量（有上限） | 普通用户，日常聊天 |
| **按量付费** | 用多少 Token 付多少钱 | 开发者，集成到产品里 |

> 详细价格对比见第 08 章。

### API 的优势

- **简单**：注册即用，不需要任何硬件
- **灵活**：可以选择不同等级的模型
- **自动更新**：AI 公司负责维护和升级
- **无需技术基础**：使用网页或 App 即可

### API 的劣势

- **数据经过云端**：你的对话内容会发送到 AI 公司的服务器
- **依赖网络**：断网就用不了
- **成本不可控**：用量大时费用可能超出预期

---

## 7.3 本地部署：把 AI "搬回家"

### 什么意思

把大模型的程序文件下载到你自己的电脑上运行，不需要联网就能使用 AI。

### 需要什么条件

| 条件 | 最低要求 | 推荐配置 |
| --- | --- | --- |
| **显卡（GPU）** | 8GB 显存 | 24GB+ 显存 |
| **内存** | 16GB | 32GB+ |
| **硬盘** | 20GB 可用空间 | 100GB+ SSD |
| **技术能力** | 会安装软件 | 会用命令行 |

### 怎么部署（概览）

目前最简单的部署方式是使用 **Ollama** 这个工具：

1. 下载安装 Ollama（一行命令搞定）
2. 选择要下载的模型（如 Llama、Qwen、DeepSeek 等）
3. 开始本地对话，数据完全不出电脑

> 💡 Ollama 让本地部署变得像"安装 App"一样简单，是目前最推荐新手使用的工具。

### 本地部署的优势

- **数据隐私**：所有数据都在本地，不会上传到任何服务器
- **完全离线**：断网也能用
- **无使用限制**：想用多少次就用多少次
- **可定制**：可以针对自己的需求微调模型

### 本地部署的劣势

- **硬件成本高**：好显卡不便宜
- **技术门槛**：需要一定的计算机基础
- **模型选择有限**：只有开源模型可以本地部署
- **性能受限**：家用硬件跑不了最大的模型

---

## 7.4 四种典型场景的选择建议

| 场景 | 推荐方式 | 原因 |
| --- | --- | --- |
| **个人日常使用** | API 订阅制 | 简单、便宜、不需要硬件 |
| **公司办公集成** | API 按量付费 | 可以接入公司系统，按需付费 |
| **敏感数据处理** | 本地部署 | 数据不出门，满足合规要求 |
| **学习和研究** | 本地部署 | 可以自由实验、微调模型 |

---

## 7.5 隐私与安全注意事项

无论选择哪种方式，都要注意：

### 使用 API 时

- **不要发送敏感信息**：密码、身份证号、银行卡号等绝对不要发给 AI
- **了解平台隐私政策**：有的平台会用你的对话数据来改进模型
- **公司数据要谨慎**：商业机密、客户信息可能需要走内部专用通道

### 本地部署时

- **配置访问控制**：不要让局域网内的人都能访问你的模型
- **定期更新**：及时修补安全漏洞
- **备份重要数据**：硬件故障可能导致数据丢失

> 💡 **最简单的原则**：把 AI 当成一个"聪明的陌生人"——你不会告诉陌生人的事，也不要告诉 AI。

---

## 7.6 未来趋势：两者会融合

目前，API 和本地部署的界限正在模糊化：

| 趋势 | 说明 |
| --- | --- |
| **端侧 AI** | 手机、电脑上预装的小型 AI，不需要联网 |
| **混合部署** | 简单任务本地处理，复杂任务上传云端 |
| **私有云** | 企业在自己的服务器上部署，兼顾自主和性能 |

> 💡 未来你可能不需要选择——设备会自动判断：哪些任务本地做，哪些任务上传做。

---

## 本章小结

| 核心要点 | 一句话记住 |
| --- | --- |
| API | 通过网络"租用"AI，简单方便 |
| 本地部署 | 把 AI "搬回家"，自主可控 |
| 个人用户 | 优先 API 订阅制，最省心 |
| 敏感数据 | 优先本地部署，数据不出门 |
| 隐私原则 | 不要告诉 AI 你不会告诉陌生人的事 |
