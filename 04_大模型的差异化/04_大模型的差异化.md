# 第 04 章 大模型的差异化

> 市面上的 AI 看起来都差不多——都是一个对话框，为什么有的贵、有的便宜？有的擅长写代码、有的擅长画图？这一章帮你搞清楚它们的区别。

---

## 4.1 通用模型 vs 垂直模型

### 通用模型

像一个"全才"，什么都能聊，但没有一个领域特别深入。

| 模型 | 出品方 | 特点 |
| --- | --- | --- |
| GPT-5.5 | OpenAI | 推理深度最强，支持计算机控制，1M+上下文 |
| Claude Opus 4.8 | Anthropic | 编码能力最强，1M上下文，多智能体协调 |
| Gemini 3.1 Pro | Google | 多模态理解最强，超长上下文 |
| DeepSeek-V4 | 深度求索 | 已进入第一梯队，性价比极高 |
| 通义千问 Qwen3.7-Max | 阿里 | 智能体原生设计，可自主执行35小时长周期任务 |
| GLM-5.1 | 智谱AI | 代码能力突出，智能体能力领先 |
| Kimi K2.5 | 月之暗面 | 200万token超长上下文 |
| 文心一言 5.0 | 百度 | 搜索增强，政务教育场景优势 |

### 垂直模型

专注某一个行业或任务，在那个领域做得更专业。

| 领域 | 代表产品 | 优势 |
| --- | --- | --- |
| 法律 | 幂律智能、法智易 | 熟悉法条和案例 |
| 医疗 | 讯飞医疗、丁香医生 AI | 医学知识更准确 |
| 金融 | BloombergGPT | 金融数据分析 |
| 代码 | GitHub Copilot、CodeBuddy | 专注编程辅助 |
| 设计 | Midjourney、GPT Image | 专注图像生成 |

> 💡 **怎么选？** 日常使用选通用模型，专业场景可以考虑垂直模型或在通用模型上做针对性微调。

---

## 4.2 闭源模型 vs 开源模型

这是 AI 领域最重要的一条分界线。

### 闭源模型

模型的代码和训练细节不公开，只能通过官方渠道使用。

| 优点 | 缺点 |
| --- | --- |
| 通常性能更强 | 必须通过网络调用 |
| 持续自动更新 | 不能自己修改或定制 |
| 使用简单，开箱即用 | 数据隐私受平台政策约束 |

代表：GPT-5、Claude Opus、文心一言 5.0

### 开源模型

模型的代码和权重公开，任何人都可以下载、修改、部署。

| 优点 | 缺点 |
| --- | --- |
| 可以本地部署，数据不出门 | 通常需要一定的技术能力 |
| 可以自由修改和定制 | 性能可能略低于顶级闭源 |
| 成本可控（甚至免费） | 需要自己准备硬件 |

代表：DeepSeek-V4、Llama 4（Meta）、Qwen3.7（阿里）、GLM-5（智谱AI）

### 对比总结

| 维度 | 闭源 | 开源 |
| --- | --- | --- |
| 性能 | 通常略高 | 差距在缩小 |
| 隐私 | 数据经过平台服务器 | 可完全本地化 |
| 成本 | 按量/订阅付费 | 硬件一次性投入 |
| 易用性 | 简单，注册即用 | 需要技术基础 |
| 定制性 | 不能改 | 随便改 |

---

## 4.3 国内模型 vs 国外模型

| 维度 | 国外模型 | 国内模型 |
| --- | --- | --- |
| 代表 | GPT-5、Claude Opus、Gemini 3 | DeepSeek、文心、通义、豆包、GLM、Kimi |
| 英文能力 | 更强 | 一般 |
| 中文能力 | 好 | 更好（更懂中文语境和文化） |
| 复杂推理 | 略领先 | 追赶中，差距不大 |
| 访问方式 | 需要科学上网 | 直接访问 |
| 价格 | 偏贵 | 便宜，甚至免费 |
| 合规性 | 数据可能出境 | 符合国内数据合规要求 |

> 💡 **结论**：日常使用、国内办公场景，优先选国产模型，方便、便宜、合规。需要英文或顶级推理能力时，可以考虑国外模型。

---

## 4.4 大模型 vs 小模型

不是所有模型都越大越好。

| 类型 | 参数量 | 优点 | 缺点 |
| --- | --- | --- | --- |
| 大模型 | 1000 亿+ | 能力强、推理准 | 贵、慢、需要大硬件 |
| 中模型 | 100-1000 亿 | 平衡性能和成本 | 复杂任务可能不够 |
| 小模型 | 70 亿以下 | 快、便宜、可手机运行 | 能力有限 |

**小模型的实际用途**：
- 手机上的离线 AI（不需要联网）
- 实时翻译、语音识别
- 简单的文本分类和摘要

> 💡 **类比**：大模型像一台旗舰级台式机，小模型像一部轻便手机——不同场景各有所需。

---

## 4.5 不同模型的"性格"

即使是同样能力的模型，因为训练方式不同，"性格"也不一样：

| 模型 | "性格"特点 |
| --- | --- |
| GPT-5 | 回答全面但偏长，支持计算机控制（Agent） |
| Claude Opus | 谨慎保守，注重安全，编码和长文写作突出 |
| 文心一言 5.0 | 中文流畅，搜索增强，有时偏“官方”语气 |
| DeepSeek-V4 | 直来直去，回答简洁，性价比极高 |
| GLM-5 | 代码能力强，智能体体验好 |
| Kimi | 超长文档处理专家 |

> 💡 **建议**：多试几个模型，找到你喜欢的"风格"，就像选一个合拍的助手。

---

## 本章小结

| 核心要点 | 一句话记住 |
| --- | --- |
| 通用 vs 垂直 | 通用什么都能做，垂直某个领域更专业 |
| 闭源 vs 开源 | 闭源简单好用，开源自由可控 |
| 国产 vs 国外 | 日常用国产就够了，便宜又方便 |
| 大模型 vs 小模型 | 大模型能力强但贵，小模型轻便但能力有限 |
| 性格差异 | 不同模型风格不同，多试几个找合拍的 |
