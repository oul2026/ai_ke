# 第 02 章 大模型的差异化

## 五大差异维度

| 维度 | 差异点 |
| --- | --- |
| 用途 | 通用模型覆盖广；垂直模型专攻特定领域（法律、医疗、代码） |
| 开放性 | 闭源只能用；开源可下载、本地部署、二次开发 |
| 地域 | 国内中文好、合规易、访问稳；国外英文强、前沿快 |
| 规模 | 大参数能力强但慢贵；小参数快便宜但上限低；MoE 折中 |
| 输入输出 | 文本最成熟；图片/语音/视频/文件支持程度不同 |

## 通用 vs 垂直

| 类型 | 特点 | 代表场景 |
| --- | --- | --- |
| 通用模型 | 训练数据广，什么都能聊 | 邮件、总结、翻译、学习 |
| 垂直模型 | 针对行业微调，术语准、流程熟 | 合同审查、病历分析、代码生成 |

**关键差异**：通用模型"都会一点"；垂直模型在特定任务上更专业，但场景窄。

## 闭源 vs 开源

| 维度 | 闭源 | 开源 |
| --- | --- | --- |
| 使用方式 | 网页/App/API，厂商维护 | 下载权重，自己部署 |
| 数据控制 | 受平台政策约束 | 可完全本地化 |
| 成本结构 | 订阅或按量付费 | 硬件+运维成本 |
| 定制能力 | 受平台限制 | 可微调、改架构 |

**关键差异**：闭源省心但受限；开源自由但需技术能力。

## 国内 vs 国外

| 维度 | 国内模型 | 国外模型 |
| --- | --- | --- |
| 中文能力 | 本土知识、公文、协作工具优化 | 英文为主，中文可能弱 |
| 合规与访问 | 数据不出境、支付方便、访问稳定 | 部分需翻墙、数据可能出境 |
| 前沿能力 | 迭代快，部分能力追平 | 顶级推理、复杂代码仍有优势 |

**关键差异**：中文办公选国内；英文和前沿任务可测国外。

## 规模差异

参数量是模型容量的上限，但容量大不代表实际表现好：

| 规模 | 特点 | 适用场景 |
| --- | --- | --- |
| 大参数（70B+） | 推理强、慢、贵 | 复杂任务、专业分析 |
| 中参数（7B-70B） | 平衡 | 日常办公、通用任务 |
| 小参数（<7B） | 快、便宜、可本地跑 | 分类、抽取、轻量任务 |
| MoE 架构 | 总参数大但只激活部分 | 兼顾能力和成本 |

**关键差异**：同一厂商的大参数模型通常强于小参数，但不同厂商之间不能只看参数——数据质量、架构、后训练都影响实际表现。

## 输入输出类型差异

大模型按输入输出模态可分为以下类型：

| 类型 | 输入 | 输出 | 代表产品 | 典型用途 |
| --- | --- | --- | --- | --- |
| 语言大模型（LLM） | 文本 | 文本 | GPT-4o、Claude、DeepSeek、Qwen | 对话、写作、翻译、代码、推理 |
| 多模态大模型 | 文本 + 图像（+ 音频） | 文本（+ 图像） | GPT-4o、Gemini、Qwen-VL | 图文理解、文档分析、视觉问答 |
| 图像生成模型 | 文本（+ 参考图） | 图像 | Midjourney、DALL·E 3、Stable Diffusion、FLUX | 插画、海报、概念设计 |
| 音频 / TTS 模型 | 文本 / 音频 | 音频 / 文本 | ElevenLabs、Fish Audio、ChatTTS、Whisper | 语音合成、语音识别、声音克隆 |
| 视频生成模型 | 文本 / 图像 | 视频 | Sora、Runway Gen-3、Pika、Kling | 短视频、动画、广告素材 |
| 代码大模型 | 文本 / 代码 | 代码 | Cursor、GitHub Copilot、CodeLlama | 代码生成、补全、审查、调试 |
| 嵌入模型（Embedding） | 文本 / 图像 | 向量（数字数组） | text-embedding-3、BGE、Jina Embeddings | 语义搜索、RAG 检索、相似度匹配 |
| Agent / 工具调用模型 | 文本 + 工具反馈 | 文本 + 工具调用指令 | GPT-4o、Claude（Tool Use）、Qwen-Agent | 自主任务规划、多步工具调用、自动化工作流 |

**关键差异**：

- **语言大模型**是当前最成熟、应用最广的类型，绝大多数 AI 办公场景以其为核心。
- **多模态大模型**在语言模型基础上扩展了感知能力，可处理图文混合输入，但生成能力仍以文本为主。
- **图像、音频、视频生成模型**各自独立，专攻单一模态的内容生产，通常需与语言模型配合完成完整工作流。
- **代码大模型**针对编程语言微调，在代码任务上显著优于通用语言模型，是开发者的核心工具。
- **嵌入模型**不生成自然语言，而是将内容转化为向量，是搜索、RAG、推荐系统的基础组件。
- **Agent 模型**以语言模型为内核，增加了工具调用与任务规划能力，是实现 AI 自动化的关键形态。
- 部分产品（如 GPT-4o）已整合多种模态于同一模型，但各模态的成熟度不均，文本能力仍显著领先。

## 使用方式差异

| 方式 | 特点 | 适合 |
| --- | --- | --- |
| 网页/App 订阅 | 直接用，厂商维护 | 普通用户、个人日常 |
| API | 程序调用，按量计费 | 开发者、企业集成、自动化 |
| 本地部署 | 自有硬件运行 | 隐私敏感、离线、研究 |

**API vs 本地部署**：

| 维度 | API | 本地部署 |
| --- | --- | --- |
| 数据控制 | 发送到云端 | 可不出本地 |
| 成本结构 | 按量付费，用量大时贵 | 硬件初始成本高，长期可能省 |
| 维护责任 | 厂商负责 | 自己负责 |
| 网络依赖 | 需要网络 | 可离线 |

**关键差异**：敏感数据（密码、客户资料、内部代码、医疗记录）不要输入公共 AI 产品，需本地部署或企业专用通道。
