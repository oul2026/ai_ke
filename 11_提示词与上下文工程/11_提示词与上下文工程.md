# 第 11 章 提示词与上下文工程

> 同样一个 AI，会问问题的人和不会问问题的人，得到的结果天差地别。这一章教你怎么"问对问题"。

---

## 11.1 什么是提示词

**提示词（Prompt）** = 你对 AI 说的话。

它可以是一个问题、一段指令、一个请求。提示词写得好不好，直接决定了 AI 回复的质量。

**同一个需求，不同提示词的效果：**

| 提示词 | AI 可能的回复 |
| --- | --- |
| "帮我写邮件" | 写一封很泛的邮件，可能需要反复修改 |
| "帮我写一封邮件，发给合作方李总，主题是确认下周二的会议时间和地点，语气友好专业，200 字以内" | 直接生成一封几乎可以直接发送的邮件 |

> 💡 **核心原则**：你对 AI 说得越清楚，AI 给你的结果越接近你想要的。

---

## 11.2 好提示词的四个要素

### 1. 角色（Role）

告诉 AI "你是谁"，让它的回答更有针对性。

- "你是一个资深营销专家……"
- "假装你是一位有 10 年经验的英语老师……"
- "以产品经理的视角……"

### 2. 任务（Task）

明确说清楚你要做什么。

- ❌ "帮我看看这个"
- ✅ "帮我分析这段用户反馈，找出用户最不满意的 3 个方面"

### 3. 上下文（Context）

给 AI 足够的背景信息。

- ❌ "写个项目总结"
- ✅ "我们团队做了一个为期 3 个月的 App 改版项目，目标是提升用户留存率。最终留存率从 30% 提升到 45%，但日活下降了 10%。帮我写一份项目总结。"

### 4. 格式（Format）

说明你希望 AI 以什么形式回复。

- "用表格形式列出"
- "分点列出，每点不超过 50 字"
- "写成正式的商务邮件格式"
- "用 Markdown 格式输出"

---

## 11.3 实用提示词模板

### 写作类

```
你是一个 [角色]。请帮我写一篇 [文体]，主题是 [主题]。
目标读者是 [读者群体]。
要求：
- 字数：约 [X] 字
- 语气：[正式/轻松/专业]
- 结构：[总分总/问题-方案/对比分析]
```

### 分析类

```
请分析以下 [内容类型]，告诉我：
1. 核心结论是什么
2. 有哪些关键数据
3. 有什么潜在问题或风险
请用表格形式呈现。

[粘贴内容]
```

### 翻译类

```
请将以下文字翻译成 [目标语言]。
要求：
- 保持原文的 [语气/风格]
- 专业术语保留英文并在括号中注明
- 不要逐字翻译，要通顺自然

[粘贴内容]
```

### 学习类

```
请用通俗易懂的语言解释 [概念]。
要求：
- 用一个生活中的比喻来辅助理解
- 给出一个具体的例子
- 最后用一句话总结
```

## 11.4 上下文工程：不只是第一句话

提示词不只是你发出的第一条消息——**整个对话的历史**都是 AI 的"上下文"。

### 上下文的作用

| 作用 | 说明 |
| --- | --- |
| **保持连贯** | AI 记得前面聊了什么，可以追问和补充 |
| **传递信息** | 前面给出的背景信息，后面不需要重复 |
| **纠正方向** | 如果 AI 跑偏了，你可以说"不对，我想要的不是这个" |

### 上下文窗口管理

回顾第 02 章：AI 的"短期记忆"是有限的。对话太长，早期内容可能被"遗忘"。

**应对策略**：
- **重要信息重复说**：在关键请求中，把重要背景再提一遍
- **适时开新对话**：一个话题聊完了，开个新对话更清晰
- **结构化你的输入**：把长内容分段给 AI，每次给一段

---

## 11.5 高级技巧

### 技巧一：思维链（Chain of Thought）

让 AI "一步步想"，回答更准确。

> "请一步一步分析这个问题，先列出已知条件，再推导结论。"

**效果**：在数学题、逻辑推理等场景中，准确率可提升 20-30%。

### 技巧二：少样本学习（Few-shot）

给 AI 几个例子，让它"照着做"。

```
请按以下格式整理用户反馈：

例子：
用户说："这个 App 打开太慢了"
分类：性能问题
情感：负面

现在请处理：
用户说："功能挺全的，就是界面有点丑"
```

### 技巧三：角色扮演深入化

不只说"你是一个专家"，而是给出更丰富的设定：

> "你是一个有 15 年经验的 UX 设计师，曾在腾讯和字节跳动工作过，擅长 B 端产品设计。请从你的角度评审以下设计方案……"

### 技巧四：分步请求

复杂任务不要一次说完，分步让 AI 处理：

1. "先帮我列出这篇文章的大纲"
2. （确认大纲后）"好的，请展开写第一部分"
3. （确认第一部分后）"继续写第二部分，风格和第一部分保持一致"

### 技巧五：让 AI 自我检查

> "请检查你刚才的回答，有没有事实错误或不一致的地方？如果有请修正。"

---

## 11.6 常见提示词错误

| 错误 | 后果 | 改进 |
| --- | --- | --- |
| 太模糊 | AI 猜测你想要什么，经常猜错 | 具体说明任务和目标 |
| 一次问太多 | AI 顾此失彼，回答质量下降 | 拆分成多个小问题 |
| 没有给背景 | AI 缺乏上下文，回答太泛 | 说明背景、受众、目的 |
| 不指定格式 | AI 自由发挥，可能不是你想要的 | 明确说"用表格""分 3 点" |
| 从不追问 | 第一次结果不满意就放弃 | 说"换个角度""更简洁一些" |

## 11.7 提示词速查表

| 你想要的 | 加上这句话 |
| --- | --- |
| 更简洁 | "用 100 字以内回答" |
| 更详细 | "请详细展开，包括原因和例子" |
| 更专业 | "以 [领域] 专家的角度回答" |
| 更通俗 | "用小学生能听懂的方式解释" |
| 更有条理 | "用编号列表分点说明" |
| 更有创意 | "给出 5 个不同的方案，越有创意越好" |
| 更准确 | "请一步步分析，并标注你的不确定性" |
