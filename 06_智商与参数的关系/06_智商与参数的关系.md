# 第 06 章 智商与参数的关系

> "参数越大越聪明"——这句话对了一半。这一章帮你搞懂大模型的"智商"到底由什么决定。

---

## 6.1 参数是什么

回顾第 02 章的内容：参数是模型内部用来"存储规律"的数字。

简单类比：

| 参数量 | 相当于 |
| --- | --- |
| 7B（70 亿） | 一个小型图书馆的藏书量 |
| 70B（700 亿） | 一个大型图书馆的藏书量 |
| 175B（1750 亿） | 几十座大型图书馆的藏书量 |

直觉上，书读得越多，知道得越多，也就越"聪明"。

**但现实并非如此简单。**

---

## 6.2 为什么"参数越大越聪明"是误区

### 反例一：小模型打败大模型

- **DeepSeek-V4**（约 1T 参数，MoE 架构激活 32B）在多个基准测试中媲美 **GPT-5.5**，但价格只有 1/27
- **GLM-5**（744B 参数，MoE 架构激活 40B）代码能力跻身全球第一梯队

### 反例二：大模型也可能很"笨"

- 参数很大但训练数据质量差的模型，表现可能不如参数小但训练精细的模型
- 就像一个读了几万卷书但从不做题的人，不一定比读了一千卷书但反复练习的人考得好

### 真正影响"智商"的因素

| 因素 | 说明 | 类比 |
| --- | --- | --- |
| **参数量** | 模型的"脑容量" | 图书馆的大小 |
| **训练数据** | 喂给模型的"书"的质量和数量 | 读的书好不好、够不够多 |
| **训练方法** | 怎么训练的（算法、策略） | 学习方法好不好 |
| **后训练** | 微调和对齐的质量 | 毕业后的岗位培训 |
| **架构设计** | 模型的"大脑结构" | 思维方式是否高效 |

> 💡 **一句话总结**：参数是必要不充分条件——太少了肯定不行，但多了不一定更好。

---

## 6.3 参数对你的实际影响

作为普通用户，你不需要关心具体参数量，只需要关注以下几点：

### 参数量大的模型

| 优势 | 劣势 |
| --- | --- |
| 复杂推理更强 | 回复速度慢 |
| 知识面更广 | 使用成本高 |
| 不容易出错 | 硬件要求高 |

### 参数量小的模型

| 优势 | 劣势 |
| --- | --- |
| 响应速度快 | 复杂任务容易出错 |
| 成本低 | 知识面有限 |
| 可本地运行 | 创意和推理弱 |

---

## 6.4 "智商"评测怎么看

业界通过多种评测来衡量模型的"智商"，但每个评测都有局限性：

| 评测 | 侧重 | 局限 |
| --- | --- | --- |
| MMLU | 综合知识广度 | 选择题形式，和实际使用有差距 |
| GSM8K | 数学推理 | 只测数学，不代表全面能力 |
| HumanEval | 编程 | 只测简单函数，不代表工程能力 |
| LMSys Arena | 真人盲测 | 最接近实战，但排名会波动 |

### 怎么参考评测结果

1. **不要只看一个分数**：不同评测侧重点不同
2. **优先看 Arena 排名**：真人对比最接近真实体验
3. **关注你需要的能力**：如果你主要写中文，看 C-Eval；如果你写代码，看 HumanEval
4. **自己试用**：评测只是参考，实际体验最重要

---

## 6.5 Scaling Law：越大越好？有边际效应

学术界有一个概念叫 **Scaling Law（缩放定律）**：

- 一般来说，参数越多、数据越多、训练越久，模型表现越好
- **但是**，增长不是线性的——翻倍参数不等于翻倍智商

### 边际效应

| 从 X 到 Y | 提升幅度 |
| --- | --- |
| 10 亿 → 100 亿参数 | 大幅提升（质变） |
| 100 亿 → 1000 亿参数 | 明显提升 |
| 1000 亿 → 1 万亿参数 | 提升有限（边际递减） |

> 💡 **类比**：一个学生从 0 分复习到 60 分很容易，从 60 分提到 90 分需要更多努力，从 95 分提到 98 分极其困难。

---

## 6.6 未来趋势：不只是"变大"

大模型的发展方向不再只是单纯增加参数，而是：

| 方向 | 说明 |
| --- | --- |
| **更高效的架构** | 用更少的参数达到同样的效果 |
| **混合专家（MoE）** | 只激活部分参数，节省算力 |
| **专项优化** | 在特定领域做到更精 |
| **多模态融合** | 同时处理文字、图像、语音 |
| **小型化** | 在手机、PC 上本地运行 |

> 💡 **DeepSeek-V4 的 MoE 架构**就是一个典型例子：总参数约 1T，但每次推理只激活约 32B，既有大模型的智商，又有小模型的速度和价格。GLM-5 也采用类似架构（744B 总参数，激活 40B）。

---

## 本章小结

| 核心要点 | 一句话记住 |
| --- | --- |
| 参数不是唯一 | 参数量大不等于更聪明 |
| 关键因素 | 参数 + 数据 + 训练方法 + 后训练，缺一不可 |
| 边际效应 | 参数翻倍不等于智商翻倍 |
| 选模型建议 | 不要看参数量，看实际评测和自己试用 |
| 未来趋势 | 不只是变大，而是变高效 |
