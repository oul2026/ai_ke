# 第 03 章 能力和价格

信息基准：2026-06-02。梯队参考公开榜单、官方模型页和官方价格页；价格变化很快，采购或接入前必须再查一次官网。

价格口径：除特别说明外，均为 API 标准实时调用价，单位是“每 100 万 Token”。价格列按“输入、输出（不思考）、缓存、思考模式（1:1总价）”展示。“输出（不思考）”只表示最终答案/模型响应 token 单价；“思考模式（1:1总价）”不是另一套固定套餐价，而是成本场景：假设每 100 万可见输出 token 同时生成 100 万 thinking/reasoning/CoT token，列出输出侧总费用。多数厂商把 thinking token 按输出 token 计费，所以同等数量时，思考模式输出侧费用通常是不思考输出费用的 2 倍。计算公式：`不思考输出侧费用 = 可见输出 token × 输出单价`；`思考模式输出侧费用 = 可见输出 token × 输出单价 + 思考 token × 思考单价`。例：Claude Opus 4.8 不思考输出 100 万 token 是 ￥170.5；若同时生成 100 万 thinking token，思考模式输出侧费用是 `170.5 + 170.5 = ￥341.0`。实际 thinking token 数量由请求决定，最终账单以 API `usage` 为准。官网没有公开思考价、模型不支持、或本轮没有核到公开价格表时，不写具体估算价。美元价格已按 2026-06-02 人民币汇率中间价 `1 USD = 6.8187 CNY` 折算为人民币，保留一位小数；低于 0.1 元的缓存价保留三位小数。思考 token 还会挤占 `max_tokens` / `max_output_tokens` 预算。缓存、批量、长上下文、区域部署、搜索工具会改变实际费用。分段计费模型默认列低输入长度档；官网明确标注限时折扣的，表格同时列标准价和当前折扣价。同一模型在官方直连、中国区云平台、国际平台价格不同，以表格标注的平台为准。

## 成本与预算

Token 是模型计费和上下文容量的基本单位。

| 文本 | 粗略估算 |
| --- | --- |
| 英文 | 1 token 约 4 个字符 |
| 中文 | 1 个汉字约 1.5 到 2 个 token |
| 1000 字中文 | 约 1500 到 2000 token |

单次 API 成本按输入、输出、缓存、思考 token 分开算：

```text
单次费用 = 输入 token × 输入单价
       + 可见输出 token × 输出单价
       + 思考 token × 思考单价
       - 缓存命中节省
       + 搜索/工具/重试等额外费用
```

预算时不要只看单价，要按业务量放大：

| 步骤 | 要算什么 |
| --- | --- |
| 1 | 单次输入 token：提示词、历史对话、上传材料 |
| 2 | 单次输出 token：最终回答长度 |
| 3 | 思考 token：开启 reasoning/thinking/CoT 后的隐藏输出 |
| 4 | 调用次数：每日调用量、并发、失败重试 |
| 5 | 折扣条件：缓存、Batch/Flex、限时折扣、区域平台 |
| 6 | 月度预算：单次费用 × 日调用量 × 使用天数 |

价格页至少核对这些字段：

| 字段 | 用途 |
| --- | --- |
| 模型名 | 防止同系列不同档位混淆 |
| 输入价 | 读材料、读历史对话的成本 |
| 输出价 | 生成回复的成本 |
| 思考价 | reasoning/thinking/CoT 是否按输出 token 收费 |
| 缓存价 | 固定系统提示、长材料复用能省多少 |
| 上下文长度 | 是否能处理长文档 |
| 限速/并发 | 是否满足业务量 |
| 折扣期限 | 防止按促销价做长期预算 |

控制成本的有效做法：

| 做法 | 原因 |
| --- | --- |
| 简单任务用轻量模型 | 降低单价 |
| 控制输出字数 | 输出通常比输入贵 |
| 只传必要上下文 | 减少输入 token |
| 固定系统提示 | 提高缓存命中 |
| 批量处理 | 降低重复输入和调度成本 |
| 先小样本测试 | 避免大规模错误调用 |

## 选型规则

1. 高风险任务先看能力，再看价格：合同、法律、医疗、代码架构、客户数据不能只按便宜选。
2. 高频低风险任务先看价格和吞吐：分类、抽取、格式转换、短文本改写可以用轻量档。
3. 中文任务不要默认国外模型更好：国内模型在中文、价格、访问、合规上经常更实用。
4. 不要只看排行榜：同一模型在写作、代码、长文、多模态、Agent 上表现可能不同。
5. 最终必须实测：同一组真实任务，记录质量、速度、成本、失败率。
